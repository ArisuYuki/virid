# 🌌 virid

**The Sovereign Logic Engine & Reactive Projection System** **解构 UI 主权，重塑逻辑边界。**

`virid` 是一个受 Rust Bevy ECS 与 NestJS 启发的轻量级消息驱动引擎。它旨在将业务逻辑从 UI 框架的“附庸”地位中解放出来，构建一个确定性、可测试且跨平台的核心。

## 🧠 核心架构：逻辑主权

在 `virid` 的世界观里，业务逻辑是第一公民，而 UI 仅仅是逻辑的**“临时投影”**，这种架构称为CCS。

- **逻辑自洽**：核心业务运行在 `@virid/core` 中，完全脱离 DOM/BOM，支持跨端复用。
- **物理隔离**：UI 框架（Vue）被降级为纯粹的状态投影仪，丧失对状态的直接修改权。
- **因果驱动**：所有的状态变更必须通过 `Message` 申请，由 `System` 裁决，形成闭环的因果链条。

## 📦 模块组成

| **模块**          | **定位**             | **关键特性**                                       |
| ----------------- | -------------------- | -------------------------------------------------- |
| **`@virid/core`** | **大脑 (Logic Hub)** | 确定性 Tick 机制、双缓冲消息池、依赖注入。         |
| **`@virid/vue`**  | **神经元 (Adapter)** | 响应式投影 (Deep Shield)、依赖系留、生命周期桥接。 |

## 🎯 为什么选择 virid?

1. **极高的一致性**：基于游戏级调度引擎，规避了复杂 UI 联动中的逻辑抖动。
2. **极低甚至零 Mock 测试**：业务逻辑可以在纯 JS 环境中通过消息流完成 100% 覆盖测试。
3. **极度安全的数据流**：通过只读投影护盾，强制单向数据流，逻辑变更 100% 可审计。
